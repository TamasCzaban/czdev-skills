# LinkedIn Content — Formatting & Style Reference

This is a **reference skill**. It defines the rules all CZ Dev LinkedIn posts must follow.
When writing posts, the `linkedin-post` skill loads this for its rules — do not invoke this skill directly to draft posts.

---

## Post Anatomy

```
HOOK (first 1-2 lines)          ← Visible before "...see more" — ≤210 chars
...see more ─────────────────── ← The click gate

BODY (story/value)
One sentence per line.
Blank lines between paragraphs.

CTA (last 1-2 lines)            ← Ask for engagement

#hashtags (3-5)
```

---

## Character Limits

| Element | Limit |
|---------|-------|
| Post text | 3,000 characters |
| Visible before "see more" | ~210 characters (~2 lines on mobile) |
| Hashtags | 3–5 recommended |
| Comment | 1,250 characters |

**The first 210 characters are everything.** If the hook fails, nobody clicks "see more."

---

## Hook Formulas

| Formula | Example |
|---------|---------|
| Contrarian opinion | "Airtable is not a CRM. It's a spreadsheet with better UI." |
| Personal story opening | "A founder called us last month. Her team was spending 3 hours a day copy-pasting between tools." |
| Surprising stat/finding | "We asked 12 founders to describe their data flow. It took an average of 35 minutes." |
| List promise | "5 signs you've outgrown Airtable (and what comes next)." |
| Bold statement | "Your no-code setup isn't the problem. Staying on it too long is." |
| Before/after | "Six months ago: rental contracts tracked in Google Sheets. Today: automated reminders, zero manual updates." |

### Hook failures to avoid

```
❌ "Excited to announce that we are pleased to share..."
❌ "In today's rapidly evolving landscape..."
❌ "I'd like to take a moment to..."
❌ Starting with a hashtag or emoji
❌ Generic wisdom: "Hard work pays off", "Innovation requires courage"
```

---

## Formatting Rules

| Rule | Reason |
|------|--------|
| One sentence per line | Mobile readability |
| Blank lines between paragraphs | Visual breathing room |
| No walls of text | Nobody reads them on mobile |
| Bold sparingly | Draws eye to key points |
| Numbered lists for tips | Scannable, shareable |
| No links in post body | LinkedIn penalizes reach — put links in first comment |

## Stop-Slop: Banned Patterns

Remove these from every post before saving:

**Punctuation:**
- Em dash (—) anywhere — restructure the sentence instead

**Banned openers:**
- "Here's the thing / Here's what / Here's why"
- "Most [founders/people] don't..."
- Wh- starters: What, When, Where, Which, Who, Why, How as the first word

**Banned structures:**
- "Not X. Y." / "Not X, it's Y." — state Y directly instead
- "None of these are..." / negative listing
- "The question is whether..." / vague declaratives
- Three-item lists — use two or four

**Corporate openers (always banned):**
- "Excited to announce"
- "I'm proud to share"
- "In today's world / landscape"
- "Leveraging synergies"

## Algorithm Signals

| Signal | Impact | How to use |
|--------|--------|------------|
| Dwell time | Very High | Write longer posts people read fully |
| Comments | Very High | Ask questions, provoke discussion |
| Saves | High | Actionable, reference-worthy content |
| "See more" clicks | High | Strong hook |
| External links | Negative | Put links in first comment |
| Editing after posting | Negative | Never edit within first hour |
| Posting frequency | Context | 3–5x/week ideal; never more than 1/day |

## CTA Formulas

End every post with one:

| Type | Example |
|------|---------|
| Question | "What does your data flow actually look like right now?" |
| Agreement check | "Hot take or obvious? Tell me below." |
| Experience ask | "Has your team hit this wall?" |
| Recommendation ask | "What would you add to this list?" |

## Posting Schedule (Budapest / CET)

Best times: **Tuesday–Thursday, 7–8 AM** — primary window.
Also valid: Monday 8 AM, Friday 7–8 AM.
Skip weekends for CZ Dev content.

**Engage in comments for 30–60 minutes after posting** — this drives reach more than the post itself.

---

## Hashtag Rotation

Pick 3–5 per post. Mix niche + one broad max.

**Niche (prefer these):** #customsoftware #foundertool #nocode #crm #automation #firebase #streamlit #b2bsaas #startuptools #buildinpublic

**Broad (max 1 per post):** #founder #startup #saas #tech #softwareengineering

---

## 80/20 Rule

80% of posts should deliver direct value (education, story, opinion, insight).
20% can reference CZ Dev services or proof points.
Never open with a promotion.
